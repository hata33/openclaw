# tasks — 功能定义

## 解决什么问题？

Agent 需要执行长时间运行的后台任务。tasks 模块提供：

1. **持久化** — 任务状态存储在 SQLite，进程重启不丢失
2. **执行管理** — 创建、启动、取消、重试
3. **状态跟踪** — 实时查询任务进度
4. **结果投递** — 完成后自动通知用户

## Task vs TaskFlow

| 特性 | Task | TaskFlow |
|------|------|----------|
| 粒度 | 单一工作单元 | 多步骤流程 |
| 状态 | pending/running/completed | 多步骤状态 |
| 子任务 | 无 | 支持子任务 |
| 等待 | 无 | 支持等待条件 |

## 存储

SQLite 存储，支持：

- 按状态查询
- 按所有者查询
- 过期清理
- 审计日志

## 投递

任务完成后将结果投递到指定渠道（Telegram/Discord 等）。
