# pairing — 策略、配置与边界情况

## 一、配对码策略

### 1.1 生成

配对码使用 `crypto.randomBytes()` 生成，确保随机性。

### 1.2 唯一性

同一用户重复请求配对时返回相同的配对码（幂等性）。

### 1.3 过期

配对码没有内置过期机制。所有者需要手动清理未使用的请求。

## 二、AllowFrom 策略

### 2.1 通配符

```yaml
allowFrom: ["*"]  # 允许所有人
```

通配符跳过配对流程，所有用户直接获得访问权限。

### 2.2 原子写入

白名单文件使用原子写入（临时文件 + rename），防止并发冲突。

### 2.3 缓存

AllowFrom 文件使用 mtime 缓存，文件未修改时不重新读取。

## 三、安全策略

### 3.1 文件锁

配对操作使用文件锁，防止多个请求同时修改白名单。

### 3.2 IP 过滤

设置码生成时排除不可路由的地址：

```
排除: loopback (127.x), RFC1918 (10.x, 172.16-31.x, 192.168.x), CGNAT
```

## 四、已知边界情况

### 4.1 多渠道配对

每个渠道有独立的 allowFrom 白名单。在一个渠道配对不影响其他渠道。

### 4.2 多账户

每个账户有独立的配对请求和白名单。

### 4.3 配置优先

如果配置中已有 allowFrom，配对系统不会覆盖配置文件中的值。
