# flows — 策略、配置与边界情况

## 一、诊断策略

### 1.1 检查分级

| 级别 | 含义 | 处理 |
|------|------|------|
| error | 必须修复 | 阻止正常使用 |
| warning | 建议修复 | 影响体验但不阻止 |
| info | 信息性 | 无需操作 |

### 1.2 自动修复

Doctor 优先提供自动修复选项，减少手动操作。

### 1.3 非交互模式

`--yes` 标志启用非交互模式，自动应用所有修复。

## 二、渠道设置策略

### 2.1 插件安装

渠道插件可能需要安装。Doctor 检测并提示安装。

### 2.2 连接测试

配置完成后自动测试连接，确保凭证有效。

## 三、已知边界情况

### 3.1 网络问题

健康检查可能因网络问题超时。检查设置合理的超时时间。

### 3.2 配置写入权限

某些修复需要写入配置文件。非交互模式下自动检查权限。
