# wizard — 功能定义与设计思想

## 这个模块解决什么问题？

OpenClaw 的配置很复杂。新用户需要配置 Gateway、模型 Provider、渠道、插件等。wizard 模块通过交互式向导简化这个过程。

## 设计思想

### 1. 两种流程

| 流程 | 说明 | 适合 |
|------|------|------|
| `quickstart` | 最少步骤，使用默认值 | 新用户快速上手 |
| `advanced` | 完整配置，所有选项可调 | 有经验的用户 |

### 2. 分步配置

向导将配置拆分为独立步骤，每步只关注一个方面：

```
Gateway → 模型 → 渠道 → 插件 → 安全
```

### 3. 密钥安全输入

`setup.secret-input.ts` 提供安全的密钥输入：

```
输入 API Key → 隐藏显示 → 验证格式 → 确认
```

### 4. 国际化

支持 3 种语言的提示信息：

```typescript
t("wizard.gateway.port.title")  // 根据语言环境返回对应文本
```

### 5. 迁移支持

`setup.migration-import.ts` 检测旧版本配置并自动导入：

```
检测旧配置文件 → 提示导入 → 迁移到新格式
```

### 6. 安全说明

`setup.security-note.ts` 在配置完成后显示安全建议：

```
⚠️ 安全提示:
- 配置文件包含敏感信息，请勿公开分享
- 建议启用 Gateway 认证
- 建议使用 SecretRef 而非明文密钥
```

### 7. 配置确认

`setup.completion.ts` 在写入前显示配置摘要，让用户确认。
