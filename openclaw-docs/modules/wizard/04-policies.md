# wizard — 策略、配置与边界情况

## 一、流程策略

### 1.1 默认值

Quickstart 流程使用合理的默认值：

```
端口: 12587
绑定: loopback
认证: token（自动生成）
```

### 1.2 渐进式

从简单到复杂：先 Gateway，再模型，最后可选的渠道和插件。

## 二、安全策略

### 2.1 密钥隐藏

密钥输入使用密码模式，不显示在终端。

### 2.2 安全说明

配置完成后显示安全建议，提醒用户注意安全。

### 2.3 不存储明文

密钥优先使用 SecretRef（$env），避免明文存储在配置中。

## 三、国际化策略

### 3.1 检测语言

根据系统语言环境自动选择界面语言。

### 3.2 支持的语言

- English (en)
- 简体中文 (zh-CN)
- 繁體中文 (zh-TW)

### 3.3 回退

未翻译的文本回退到英文。

## 四、已知边界情况

### 4.1 中断恢复

向导被中断（Ctrl+C）时不保存部分配置。用户需要重新运行。

### 4.2 已有配置

检测到已有配置时提示是否覆盖，避免意外丢失。

### 4.3 插件安装失败

插件安装可能因网络问题失败。向导显示错误但继续完成其他配置。
