# transcripts — 策略、配置与边界情况

## 一、摘要策略

### 1.1 模式匹配

使用正则表达式匹配关键词，而非 AI 摘要。优点：快速、确定性、无额外成本。

### 1.2 限制

简单模式匹配可能遗漏隐含的决策或行动项。复杂会议可能需要 AI 辅助摘要。

## 二、存储策略

### 2.1 目录命名

```
<date>-<session-id>/
```

日期前缀便于按时间排序。session-id 安全化处理（移除特殊字符）。

### 2.2 JSONL 格式

话语以 JSONL 存储，支持增量追加（实时转录场景）。

## 三、安全策略

### 3.1 路径安全

```typescript
safeSegment(value) // 移除非安全字符
```

### 3.2 文件名安全

日期段使用正则提取，确保格式正确。

## 四、已知边界情况

### 4.1 大量话语

`maxUtterances` 限制每个会话的最大话语数，防止无限增长。

### 4.2 实时转录延迟

实时转录可能因为网络延迟导致话语乱序。Provider 负责排序。

### 4.3 手动导入格式

手动导入只支持 "Speaker: Text" 格式。复杂格式需要预处理。
