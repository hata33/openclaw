# node-host — 策略

## 一、执行安全

### 白名单

预批准的命令直接执行，无需用户干预。

### 审批

不在白名单的命令需要用户审批。支持 "allow always" 持久化。

### 内联 eval 检测

检测命令中的 `$(...)`、反引号等内联执行，提高安全等级。

## 二、连接策略

### 自动重连

WebSocket 断开后自动重连，支持指数退避。

### 设备身份

使用设备证书（`loadOrCreateDeviceIdentity`）标识节点。

## 三、超时策略

所有命令执行支持超时，防止无限运行。使用 AbortController 取消。
