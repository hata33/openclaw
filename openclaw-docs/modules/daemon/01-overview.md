# daemon — 功能定义

## 解决什么问题？

OpenClaw Gateway 需要作为后台服务持续运行。daemon 模块处理跨平台的服务管理。

## 服务生命周期

```
1. 安装
   → 生成服务配置文件（unit/plist/task）
   → 注册到系统服务管理器

2. 启动
   → 通过服务管理器启动
   → 环境变量注入

3. 运行
   → 自动重启（崩溃恢复）
   → 日志管理

4. 停止/卸载
   → 优雅停止
   → 清理配置文件
```

## 环境管理

服务进程和交互式终端的环境不同。`service-env.ts` 确保：

- PATH 包含 Node.js 和 OpenClaw
- 代理设置正确
- 语言环境正确

## 审计

`service-audit.ts` 记录服务操作日志。
