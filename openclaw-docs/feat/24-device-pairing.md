# 24 — 设备配对

> OpenClaw 的设备配对模块（Pairing）实现了安全的设备信任建立流程，
> 通过挑战-响应机制将移动设备、桌面应用等配对为可信节点。

## 设计思想

```
OpenClaw 运行在服务器上，但用户有多台设备：
  → macOS 菜单栏 App
  → iOS/Android 移动端
  → 其他桌面设备

这些设备需要安全地连接到 Gateway：
  → 不能让任何设备随意连接
  → 需要用户显式授权
  → 连接后可执行受限操作
```

## 配对流程

```
┌─────────┐                    ┌─────────┐
│  设备     │                    │  Gateway │
│  (Node)  │                    │          │
└────┬─────┘                    └────┬─────┘
     │   1. 扫描二维码/输入配对码    │
     │───────────────────────────→│
     │                              │
     │   2. 发送配对请求 (pairing-challenge)
     │───────────────────────────→│
     │                              │
     │   3. 返回挑战 (challenge)    │
     │←───────────────────────────│
     │                              │
     │   4. 签名并返回响应           │
     │───────────────────────────→│
     │                              │
     │   5. 验证 → 配对成功          │
     │←───────────────────────────│
     │   (返回配对凭证)              │
```

## 核心组件

### 配对挑战

`src/pairing/pairing-challenge.ts` — 生成和验证配对挑战：

```typescript
// 生成挑战
generateChallenge()
  → 生成随机 nonce
  → 绑定设备标识
  → 设置过期时间

// 验证响应
verifyChallenge(challenge, response)
  → 检查 nonce 匹配
  → 检查签名有效
  → 检查未过期
  → 通过 → 生成配对凭证
```

### 配对消息

`src/pairing/pairing-messages.ts` — 配对协议的消息格式：

```
配对请求 → 包含设备信息、公钥
配对响应 → 包含挑战
挑战响应 → 包含签名
配对确认 → 包含凭证
```

### 允许来源存储

`src/pairing/pairing-store.ts` — 存储已配对设备的信任关系：

```
配对成功
  → 记录设备 ID、公钥、权限
  → 持久化到 pairing store
  → 后续连接通过 store 验证
```

### 配对标签

`src/pairing/pairing-labels.ts` — 为配对设备分配可读标签：

```
设备配对 → 生成标签（如 "iPhone-15", "MacBook-Pro"）
  → 用户可自定义标签
  → 用于显示和识别
```

## Allow-From 文件

`src/pairing/allow-from-store-file.ts` — 外部管理的允许列表：

```
某些场景下，配对通过外部系统管理
  → 读取 allow-from 文件
  → 文件格式：一行一个设备 ID
  → 配合 pairing store 使用
```

## DM 访问控制

配对与 DM（Direct Message）访问控制紧密关联：

```
未配对设备 → 只能通过公共渠道交互
已配对设备 → 获得额外权限：
  → DM 直接消息
  → 远程命令执行
  → 文件传输
  → 媒体捕获（摄像头/屏幕）
```

## 安全保障

1. **挑战-响应** — 防止中间人攻击
2. **签名验证** — 确保设备身份真实性
3. **过期机制** — 配对请求有时效限制
4. **权限分级** — 配对设备有不同权限级别
5. **可撤销** — 可随时解除配对

## 关键代码入口

| 文件 | 职责 |
|------|------|
| `src/pairing/pairing-challenge.ts` | 挑战生成与验证 |
| `src/pairing/pairing-messages.ts` | 配对消息格式 |
| `src/pairing/pairing-store.ts` | 配对状态存储 |
| `src/pairing/pairing-labels.ts` | 设备标签 |
| `src/pairing/allow-from-store-file.ts` | 外部允许列表 |

## 总结

1. **安全配对** — 挑战-响应机制确保设备可信
2. **凭证管理** — 配对后颁发凭证，后续连接免配对
3. **权限分级** — 不同配对级别对应不同操作权限
4. **可撤销** — 随时解除配对，立即生效
